1. **Where do Drush commands go?** (CRITICAL NEVER first)
   - src/Drush/Commands/ directory requirement (the #1 breaking change)
   - Namespace: Drupal\{module}\Drush\Commands
   - WRONG/RIGHT callout: src/Commands/ (old) vs src/Drush/Commands/ (Drush 12+). State that files in src/Commands/ are silently ignored by Drush 12+ auto-discovery.
   - Class naming: {Module}Commands extends DrushCommands

2. **Dependency injection -- AutowireTrait** (CRITICAL NEVER second)
   - `use AutowireTrait;` in class body
   - Constructor injection with type hints -- Drush resolves from service container
   - `parent::__construct()` call REQUIRED (Symfony Console initializes command name)
   - WRONG/RIGHT callout: drush.services.yml (deprecated Drush 12+, removed Drush 14) vs AutowireTrait
   - WRONG/RIGHT callout: Omitting parent::__construct() causes "LogicException: Command has no name"
   - #[Autowire] attribute for ambiguous services

3. **Command declaration -- PHP 8 attributes** (CRITICAL NEVER third)
   - `use Drush\Attributes as CLI;` import alias
   - #[CLI\Command(name: 'module:action', aliases: ['m:a'])]
   - #[CLI\Argument(name: ..., description: ...)]
   - #[CLI\Option(name: ..., description: ...)]
   - #[CLI\Usage(name: 'drush module:action arg', description: ...)]
   - WRONG/RIGHT callout: @command docblock annotations (Drush 8-11 legacy) vs PHP 8 attributes

4. **Complete Drush 12 command example (primary -- D10 compatible)**
   - Full working example from research: extends DrushCommands, #[CLI\Command], AutowireTrait, constructor with parent::__construct(), EntityTypeManagerInterface injection
   - This is the copy-paste pattern -- annotate every line

5. **Output patterns**
   - $this->io()->writeln() for simple output
   - SymfonyStyle tables for formatted output
   - RowsOfFields return type for drush table formatting
   - $this->logger()->error/warning/success for status

6. **D11/Drush 13.7+ forward-looking pattern**
   - extends Command (Symfony) instead of DrushCommands
   - #[AsCommand(name:, description:, aliases:)] instead of #[CLI\Command]
   - configure() method for arguments/options
   - execute(InputInterface, OutputInterface) returning Command::SUCCESS/FAILURE
   - Same: src/Drush/Commands/ directory, AutowireTrait, parent::__construct()
   - Full working example from research
