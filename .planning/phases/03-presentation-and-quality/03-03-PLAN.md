Key database API patterns from research:
- hook_schema(): define custom tables in .install file
- Schema field types: serial, varchar, int, text, float, numeric, blob
- Database service: \Drupal::database() or inject database connection
- Static queries: $database->query("SELECT ... WHERE [col] = :val", [':val' => $v])

- Joins: $query->join('table', 'alias', 'condition')
- INSERT: $database->insert('table')->fields($fields)->execute()
- UPDATE: $database->update('table')->fields($fields)->condition()->execute()
- DELETE: $database->delete('table')->condition()->execute()
- MERGE (upsert): $database->merge('table')->keys($keys)->fields($fields)->execute()
- Bracket syntax for column names: [column_name] protects against reserved words
- Curly braces for table names: {table_name} adds prefix

**SKILL.md body (under 500 lines):**

1. **Decision tree: "How should you access data?"**
   - Working with entities (nodes, users, taxonomy terms, custom entities)? -> Use Entity Query, NOT direct database queries
   - Need complex SQL that Entity Query cannot handle? -> Use Database API, then load entities by ID
   - Custom tables (non-entity data)? -> Define with hook_schema(), query with Database API
   - Simple one-off query? -> Static query with placeholders
   - Complex, alterable query? -> Dynamic query builder
   - WRONG-WAY CALLOUT: Using direct SQL queries for entity data. Always use Entity Query (`\Drupal::entityQuery()` or entity storage `loadMultiple()`). Direct SQL bypasses access control, field storage, and cache invalidation.

2. **Getting the database connection**
   - Inject `\Drupal\Core\Database\Connection` via DI (preferred)
   - Service name: `database`
   - Static fallback: `\Drupal::database()` (only in .module files or procedural code)
   - Show service injection pattern in a service class

3. **hook_schema() -- defining custom tables**
   - Implement in `.install` file
   - Returns array of table definitions with fields, primary key, indexes, unique keys
   - Common field types: `serial` (auto-increment), `varchar` (with length), `int` (with size: tiny/small/medium/normal/big), `text` (with size), `float`, `numeric` (with precision/scale), `blob`
   - Field properties: `type`, `length`, `size`, `not null`, `default`, `unsigned`, `description`
   - Tables are created on module install, dropped on uninstall
   - Show complete hook_schema() example from book (sports/teams table)

4. **Static queries -- simple, direct SQL**
   - `$result = $database->query("SELECT * FROM {table} WHERE [column] = :value", [':value' => $val])`
   - `{table}` adds table prefix (curly braces)
   - `[column]` protects column name against reserved words (square brackets)
   - `:value` is a named placeholder (colon prefix)
   - WRONG-WAY CALLOUT: Concatenating user input into SQL strings. ALWAYS use placeholders (`:name`). Never write `$database->query("SELECT * FROM {users} WHERE name = '$name'")`. This is a SQL injection vulnerability.
   - Result is iterable: `foreach ($result as $record) { $record->column; }`
   - `->fetchAll()`, `->fetchField()`, `->fetchAssoc()`, `->fetchCol()` for different return shapes

5. **Dynamic select queries -- complex, alterable**
   - `$query = $database->select('table', 'alias')`
   - `->fields('alias', ['col1', 'col2'])` or `->fields('alias')` for all columns
   - `->addField('alias', 'column', 'column_alias')` for individual fields
   - `->condition('column', $value, '=')` (operator defaults to =)
   - `->condition('column', [$v1, $v2], 'IN')`
   - `->condition('column', NULL, 'IS NOT NULL')`
   - `->orderBy('column', 'ASC')` or 'DESC'
   - `->range($start, $length)` for LIMIT/OFFSET
   - `->countQuery()` returns a count query
   - `->execute()` returns result set
   - Show complete dynamic select example

6. **Joins**
   - `$query->join('table', 'alias', 'alias.col = main.col')` -- INNER JOIN
   - `$query->leftJoin('table', 'alias', 'condition')` -- LEFT JOIN
   - After join, use `$query->addField('alias', 'column')` to select from joined table
   - Show join example from book (players + teams)

7. **Pager queries**
   - `$query = $database->select('table', 'alias')->extend('Drupal\Core\Database\Query\PagerSelectExtender')`
   - `->limit(10)` sets items per page
   - Add `['#type' => 'pager']` render element to output for pager links
   - Pager is automatic -- just extend and limit

8. **INSERT, UPDATE, DELETE, MERGE**
   - INSERT: `$database->insert('table')->fields(['col' => 'value'])->execute()` -- returns insert ID
   - UPDATE: `$database->update('table')->fields(['col' => 'new_value'])->condition('id', $id)->execute()` -- returns affected rows
   - DELETE: `$database->delete('table')->condition('id', $id)->execute()` -- returns affected rows
   - MERGE (upsert): `$database->merge('table')->keys(['id' => $id])->fields(['col' => 'value'])->execute()`
   - Show examples of each

9. **Query altering (hook_query_alter)**
   - `hook_query_TAG_alter(AlterableInterface $query)` -- alters tagged queries
   - Queries opt in via `$query->addTag('tag_name')`
   - Entity queries are automatically tagged (e.g., `entity_query`, `node_access`)
   - Use for access control, soft deletes, multi-tenancy
   - WRONG-WAY CALLOUT: Altering queries without checking the tag. Always verify the query is the one you intend to alter. Altering untagged or wrong-tagged queries causes subtle bugs.

10. **Update hooks for schema changes**
    - `function module_name_update_NNNN(&$sandbox)` in .install file
    - NNNN starts at 9001 for new modules (convention)
    - Use `$schema = \Drupal::database()->schema()` for DDL: `->addField()`, `->changeField()`, `->addIndex()`, `->dropTable()`
    - For batch updates: use `$sandbox` array for progress tracking
    - WRONG-WAY CALLOUT: Modifying hook_schema() without an update hook. hook_schema() only runs on install. Existing sites need update hooks to change schema.

11. **Cross-references**
    - "See also: drupal-entities-fields (if installed) for Entity Query and entity storage patterns. Use Entity Query for ALL entity data -- Database API is for custom tables only. If not available, use `\Drupal::entityQuery('entity_type')` for entity queries."
    - "See also: drupal-testing (if installed) for Kernel tests that verify database operations with `$this->installSchema()`. If not available, Kernel tests require `protected static $modules` and manual schema installation."
    - "See also: drupal-module-scaffold (if installed) for .install file patterns where hook_schema() and update hooks live."
    - "See also: drupal-caching (if installed) for cache tag invalidation when custom table data changes. Use `Cache::invalidateTags()` after writes."
