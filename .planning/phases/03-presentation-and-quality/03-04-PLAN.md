**SKILL.md body (under 500 lines):**

1. **Decision tree: "What are you testing?"**
   - Single class methods with mockable dependencies, no Drupal services? -> Unit test (UnitTestCase)
   - Component needing database, services, or config but NOT browser? -> Kernel test (KernelTestBase)
   - Page behavior, form submission, navigation, HTTP responses? -> Functional test (BrowserTestBase)
   - JavaScript interactions, Ajax callbacks, dynamic DOM? -> FunctionalJavascript test (WebDriverTestBase)
   - Rule: ALWAYS use the LOWEST level that covers your need. Faster tests = faster feedback.
   - WRONG-WAY CALLOUT: Using BrowserTestBase when KernelTestBase suffices. Functional tests install a full Drupal site per test class -- 10-100x slower than Kernel tests. Only use Functional when you need browser simulation.

2. **Test directory structure**
   ```
   modules/my_module/
     tests/
       src/
         Unit/           -> extends UnitTestCase
         Kernel/         -> extends KernelTestBase
         Functional/     -> extends BrowserTestBase
         FunctionalJavascript/  -> extends WebDriverTestBase
   ```
   - Namespace: `Drupal\Tests\my_module\{Unit|Kernel|Functional|FunctionalJavascript}`
   - File naming: `*Test.php` (must end in Test)
   - Test method naming: `test*()` (must start with test) or use `@test` annotation

3. **Unit tests (UnitTestCase)**
   - Extends `Drupal\Tests\UnitTestCase`
   - No Drupal bootstrap -- pure PHP testing
   - Use `$this->createMock()` for dependencies
   - Use `$this->getConfigFactoryStub()` for config
   - Best for: utility classes, data transformations, business logic
   - Show complete unit test example from book (Calculator)
   - Common assertions: `assertEquals`, `assertTrue`, `assertFalse`, `assertNull`, `assertCount`, `assertInstanceOf`

4. **Kernel tests (KernelTestBase)**
   - Extends `Drupal\KernelTests\KernelTestBase`
   - Boots Drupal kernel with specified modules
   - `protected static $modules = ['my_module', 'system', 'user'];` -- MUST list all required modules
   - `$this->installSchema('module', 'table_name')` -- install specific table schemas
   - `$this->installEntitySchema('entity_type')` -- install entity tables
   - `$this->installConfig(['module'])` -- install default config
   - `$this->container->get('service_name')` -- access services
   - WRONG-WAY CALLOUT: Missing modules in `$modules` array. Kernel tests only load listed modules. Missing modules cause cryptic "Service not found" or "Table not found" errors. Include ALL dependency modules.
   - WRONG-WAY CALLOUT: Forgetting `installSchema()` for custom tables. Unlike full installs, Kernel tests do NOT run hook_schema() automatically. You must call `$this->installSchema('module', 'table')` in setUp().
   - Show complete Kernel test example from book (TeamCleaner)

5. **Functional tests (BrowserTestBase)**
   - Extends `Drupal\Tests\BrowserTestBase`
   - Installs a full Drupal site with specified modules
   - `protected static $modules = ['my_module', 'user', 'node'];`
   - `protected $defaultTheme = 'stark';` -- REQUIRED in D10+
   - `$this->drupalGet('/path')` -- navigate to a page
   - `$this->drupalLogin($user)` / `$this->drupalLogout()`
   - `$this->drupalCreateUser(['permission1', 'permission2'])`
   - `$this->submitForm($edit, 'Submit button text')` -- fill and submit forms
   - `$this->assertSession()` -- returns WebAssert with chained assertions
   - Common assertions: `->pageTextContains()`, `->statusCodeEquals()`, `->linkExists()`, `->fieldExists()`, `->elementExists()`
   - WRONG-WAY CALLOUT: Missing `$defaultTheme` property. BrowserTestBase requires `protected $defaultTheme = 'stark'` (or another theme). Without it, tests fail with an unclear error.
   - Show complete Functional test example from book (HelloWorldPageTest)

6. **FunctionalJavascript tests (WebDriverTestBase)**
   - Extends `Drupal\FunctionalJavascriptTests\WebDriverTestBase`
   - Requires ChromeDriver or Selenium (environment-specific setup)
   - Same page navigation as BrowserTestBase
   - Additional: `$this->assertSession()->waitForElement()`, `waitForText()`
   - Use for: Ajax callbacks, JavaScript behaviors, dynamic DOM updates
   - Keep brief -- most modules rarely need JS tests

7. **Test registration (@group)**
   - Every test class MUST have `@group module_name` annotation
   - D10: `@group` annotation (PHPUnit 9)
   - D11: `@group` annotation still works (PHPUnit 10) -- PHP attributes also supported but `@group` is safe across both versions
   - WRONG-WAY CALLOUT: Omitting @group annotation. Without `@group`, the test will not be discovered by Drupal's test runner.

8. **setUp() pattern**
   - Always call `parent::setUp()` FIRST
   - Kernel: install schemas, entity schemas, config after parent::setUp()
   - Functional: create users, content types, test data after parent::setUp()
   - Show setUp() examples for both Kernel and Functional

9. **Data providers**
   - `@dataProvider methodName` annotation for parameterized tests
   - Provider method returns array of arrays (each inner array = one test case)
   - Reduces test duplication for input/output validation
   - Show brief data provider example

10. **Running tests**
    - `php core/scripts/run-tests.sh --module my_module` (Drupal test runner)
    - `vendor/bin/phpunit modules/custom/my_module/tests/` (PHPUnit directly)
    - `vendor/bin/phpunit --group my_module` (by group)
    - `vendor/bin/phpunit --filter testMethodName` (single test)

11. **Cross-references**
    - "See also: drupal-forms-api (if installed) for testing forms with submitForm() and form validation assertions. If not available, use `$this->submitForm(['field_name' => 'value'], 'Submit')` in BrowserTestBase."
    - "See also: drupal-entities-fields (if installed) for testing entity creation with installEntitySchema() in Kernel tests. If not available, call `$this->installEntitySchema('entity_type')` in setUp()."
    - "See also: drupal-database-api (if installed) for testing custom table operations with installSchema() in Kernel tests. If not available, call `$this->installSchema('module', 'table')` in setUp()."
    - "See also: drupal-routing-controllers (if installed) for testing route access and page responses with drupalGet(). If not available, use `$this->drupalGet('/path')` and `$this->assertSession()->statusCodeEquals(200)`."
    - "See also: drupal-caching (if installed) for testing cache invalidation behavior. If not available, verify cache tags with `$this->assertSession()->responseHeaderContains('X-Drupal-Cache-Tags', 'tag')`."
