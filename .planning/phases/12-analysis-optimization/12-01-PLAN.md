**SKILL.md content must cover these 4 patterns with WRONG/RIGHT examples:**

1. **Cuddled brace style** (caused routing-controllers failure #2):
   - WRONG: `} else {` split across lines or missing newline
   - RIGHT: `} else {` on same line, `} catch (...) {` on same line
   - Drupal standard: "cuddled" elses/catches (opening brace on same line as control keyword)

2. **Missing docblocks** (caused batch-queue-cron failure #4):
   - WRONG: Class or method without docblock
   - RIGHT: All classes get `/** * Class description. */`, all public methods get `/** * Method description. * * @param ... * @return ... */`
   - Hook implementations use: `/** * Implements hook_name(). */` (no param/return tags)
   - `@file` docblocks ONLY for procedural files (.module, .install, .theme), NOT for class files

3. **Nullable parameter types** (caused views-dev failure #6):
   - WRONG: `public function init(..., $options = NULL)` or `array $options = NULL`
   - RIGHT: `public function init(..., ?array $options = NULL)` -- use `?type` nullable syntax
   - Required for PHP 8.4+ and Drupal 11 compatibility
   - Critical when overriding parent methods that have nullable parameters

4. **General patterns:**
   - One class per file
   - Trailing commas in multi-line arrays
   - Short array syntax `[]` not `array()`
   - Single blank line between methods
   - No closing `?>` tag in PHP files

**evals/evals.json:** Create an eval that generates a Drupal module and checks phpcs compliance. The prompt should ask for a simple module with a service class, a controller, and a hook implementation to exercise docblock, brace style, and nullable type patterns. Expectations should check phpcs passes and specific patterns are present (docblocks on classes/methods, cuddled braces, nullable types where applicable).

**Patch 1: batch-queue-cron SKILL.md** (fixes failure #3 -- no try/catch in processItem)

Find the `processItem()` method example around line 228. Replace the simple throw with a try/catch pattern:

```php
public function processItem($data) {
  try {
    $id = isset($data->id) && $data->id ? $data->id : NULL;
    if (!$id) {
      throw new \Exception('Missing team ID');
    }
    $this->database->delete('teams')
      ->condition('id', $id)
      ->execute();
  }
  catch (SuspendQueueException $e) {
    // Systemic failure -- rethrow to suspend queue processing.
    throw $e;
  }
  catch (\Exception $e) {
    // Log and skip bad items (don't rethrow = item deleted from queue).
    \Drupal::logger('sports')->error('Queue processing failed: @msg', ['@msg' => $e->getMessage()]);
  }
}
```

Also add `use Drupal\Core\Queue\SuspendQueueException;` to the use statements in the same code block.
