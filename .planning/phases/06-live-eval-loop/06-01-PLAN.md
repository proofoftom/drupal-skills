  <files>eval/setup-drupal-env.sh, eval/teardown-drupal-env.sh</files>
  <action>
Create two executable shell scripts for managing isolated Drupal evaluation environments.

**eval/setup-drupal-env.sh:**
- Usage: `./eval/setup-drupal-env.sh <unique-name>`
- Copies os-knowledge-garden to `/tmp/os-kg-{name}` (local cp -a, fast)
- CRITICAL: The os-knowledge-garden .ddev/config.yaml has NO `name:` field. Use `sed -i "1a name: os-kg-${NAME}" .ddev/config.yaml` to INSERT the name field (not substitute). The research confirmed the PRD's sed command would silently fail because there is no existing `name:` line to match.
- Runs `ddev start` then `bash scripts/install.sh --demo=cascadia`
- Outputs the target directory path on success
- Uses `set -euo pipefail` for safety
- Exits with error if target directory already exists

**eval/teardown-drupal-env.sh:**
- Usage: `./eval/teardown-drupal-env.sh <unique-name>`
- Runs `ddev delete -O -y` in the target directory (suppressing errors if already gone)
- Removes `/tmp/os-kg-{name}` directory
- Exits cleanly if directory doesn't exist (idempotent)

Make both scripts executable with chmod +x.
  </action>

  </verify>
