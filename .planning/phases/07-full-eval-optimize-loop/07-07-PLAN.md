<interfaces>
<!-- Eval execution pattern (NON-NEGOTIABLE constraints) -->

1. Use Agent subagents for eval execution (NOT headless `claude -p` -- it hangs silently)
2. 1 executor agent per skill: each handles exactly 1 skill (with + without runs)
3. Sonnet 4.6 as executor model (GSD balanced profile already uses Sonnet for executors)
4. Batch skills by RAM: max 2-3 skills simultaneously (4-6 ddev instances)
5. Must teardown ddev between batches: `bash eval/teardown-drupal-env.sh <name>`
6. Must include "Do NOT ask questions -- just create the code" in without-skill prompts
7. Must delete stale os-knowledge-garden ddev projects before runs
8. Without-skill prompts must NOT reference any SKILL.md files

<!-- Iteration directory naming -->
- Skills with only iteration-1: use iteration-2
- caching + module-scaffold (have iteration-2): use iteration-3

<!-- Workspace directory structure -->
{skill}-workspace/iteration-N/eval-{name}/{with,without}_skill/run-1/{outputs/,transcript.md,grading.json}

<!-- Eval pipeline per skill -->
1. Read evals.json for skill
2. Create workspace directories
3. Setup 2 ddev envs (with-skill, without-skill) via eval/setup-drupal-env.sh
4. Run with-skill Sonnet agent: reads SKILL.md, executes prompt in ddev env
5. Run without-skill Sonnet agent: same prompt, no SKILL.md, "Do NOT ask questions" prefix
6. Copy outputs from ddev envs to workspace
7. Grade both runs against evals.json expectations (read output files, check assertions, produce grading.json with evidence)
8. Teardown both ddev envs
9. Aggregate into benchmark.json via aggregate_benchmark.py
</interfaces>
</context>

**Sub-batch A (3 skills concurrently): routing-controllers, forms-api, plugins-blocks**

For each skill, spawn an Agent subagent that:
1. Reads the rewritten evals.json from skills/drupal-{skill}/evals/evals.json
2. Creates workspace: `mkdir -p drupal-{skill}-workspace/iteration-2/eval-{name}/{with,without}_skill/run-1/outputs`

6. Grades both runs against ALL expectations in evals.json. For each expectation:
   - Read the relevant output files (PHP, YAML, etc.) from the ddev env
   - Determine pass/fail with evidence (specific file, line, content)
   - Write grading.json to workspace
7. Teardown: `bash eval/teardown-drupal-env.sh {skill}-with` and `bash eval/teardown-drupal-env.sh {skill}-without`
8. Aggregate: `python3 aggregate_benchmark.py drupal-{skill}-workspace/iteration-2 --skill-name drupal-{skill}`

Wait for all 3 agents to complete. Verify benchmark.json exists for each.
