Run all 3 skills through the eval pipeline. Process ONE skill at a time to stay within RAM limits. Same pattern as plan 07-02 Task 1.

**For each skill, repeat:**
1. Create workspace dirs: `mkdir -p ${SKILL}-workspace/iteration-1/eval-${EVAL_NAME}/{with_skill,without_skill}/run-1/outputs`
2. Copy eval_metadata.json from evals.json
3. Setup 2 ddev envs: `bash eval/setup-drupal-env.sh ${SHORT}-with` and `${SHORT}-without`
4. Run 2 headless Sonnet: `env -u CLAUDECODE claude -p --model sonnet --permission-mode bypassPermissions "[prompt]"`
5. Verify outputs exist
6. Teardown: `bash eval/teardown-drupal-env.sh ${SHORT}-with` and `${SHORT}-without`

**Skill-specific details:**

**routing-controllers** (eval-json-endpoint):
- Env names: routing-with, routing-without
- Module: api_status_endpoint
- With-skill reads: skills/drupal-routing-controllers/SKILL.md + references/
- Verify: `ddev drush en api_status_endpoint -y`

**access-security** (eval-permission-route):
- Env names: access-with, access-without
- Module: restricted_reports
- With-skill reads: skills/drupal-access-security/SKILL.md + references/
- Verify: `ddev drush en restricted_reports -y`

**theming** (eval-themed-list):
- Env names: theming-with, theming-without
- Module: featured_resources
- With-skill reads: skills/drupal-theming/SKILL.md + references/
- Verify: `ddev drush en featured_resources -y`
