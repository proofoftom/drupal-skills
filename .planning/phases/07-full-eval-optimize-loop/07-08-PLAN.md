Classify each skill into tiers:
- **High delta** (>= +30%): Skill dramatically improves output

- **Zero delta** (0%): Sonnet already knows these patterns

   - Note which NEW assertions successfully discriminated

<task type="auto">
  <name>Task 2: Iterate on weak-delta skills and finalize analysis</name>
  <files>eval/analysis-full-13.md</files>
  <action>
For each skill classified as "zero delta" or "low delta" in Task 1:

**Step 1: Diagnose root cause** -- Read grading.json evidence for both with/without runs:
- Are assertions STILL too generic? (Both pass easily despite rewrite)
- Does Sonnet genuinely know this pattern well from training data?
- Did the eval prompt fail to trigger the skill's specific wrong-way traps?

**Step 2: Determine if iteration is feasible and worth it:**

Option A -- Tighten assertions further:
- If assertions are still testing standard patterns, add MORE SPECIFIC checks
- Example: Instead of "uses DI" test for "constructor params are private readonly" or "no \\Drupal:: static calls in ANY method"
- Update the skill's evals.json with tighter assertions
- Re-run ONLY the affected skill through the eval pipeline (1 Agent, 1 ddev pair, grade, aggregate)
- Update benchmark.json

Option B -- Accept as valid finding:
- If Sonnet genuinely knows the pattern, document this in the analysis
- Note: "This skill adds value for weaker models, more ambiguous prompts, or edge cases not covered by this eval"
- Do NOT spend more than 2 iteration cycles on any single skill

**Step 3: Update eval/analysis-full-13.md with:**

8. **Iteration Log** -- For each weak-delta skill:
   - What was diagnosed
   - What was tried (assertion tightening, prompt revision, or acceptance)
   - Result of re-run (if applicable)
   - Final disposition (improved delta, or accepted valid finding)

9. **Overall Verdict** -- Support with data:
   - How many skills show >= +10% delta?
   - What is the average delta across all 13?
   - Are the high-delta skills (caching, scaffold) still high?
   - Key insight: What makes a skill effective?
   - Recommendation: Which skills are highest value, which are lowest?

10. **Recommendations**
    - Skills to prioritize for distribution (high delta, clear value)
    - Skills that may need future work (low delta but theoretically valuable)
    - Patterns for future skill design (what makes assertions effective)
    - Comparison: Sonnet vs weaker models (reference Phase 6 Opus data if available)

**CONSTRAINT:** Do NOT spend more than 2 iteration cycles total across all weak-delta skills. If a skill still shows 0% after one tightening attempt, accept it.
  </action>
