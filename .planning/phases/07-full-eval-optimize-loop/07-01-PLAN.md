<task type="auto">
  <name>Task 1: Install agent-browser and create E2E assertion helper</name>
  <files>eval/e2e-assert.sh</files>
  <action>
1. Install agent-browser globally: `npm install -g agent-browser && agent-browser install --with-deps`
2. Verify installation: `agent-browser open https://example.com && agent-browser get title && agent-browser close`
3. Create `eval/e2e-assert.sh` -- a reusable helper script that grading tasks will call to run E2E assertions.

The script should:
- Accept arguments: `e2e-assert.sh <ddev-project-name> <url-path> <assertion-type> [expected-value]`
- Construct the full URL as `https://os-kg-<ddev-project-name>.ddev.site<url-path>`
- Support these assertion types:
  - `page-contains <text>` -- open URL, get page text via `agent-browser snapshot`, check if text appears
  - `status-ok` -- open URL, verify no error page (check title is not "Page not found", page text does not contain "The requested page could not be found")
  - `status-forbidden` -- open URL, verify 403 response (page contains "Access denied" or title contains "403")
  - `element-exists <selector>` -- open URL, check that the selector matches an element via `agent-browser find`
  - `form-has-field <label>` -- open URL, verify a form field with the given label exists via `agent-browser find label`
- Output: `PASS: <assertion description>` or `FAIL: <assertion description>` on stdout
- Always run `agent-browser close` in a trap on EXIT to clean up
- Exit code: 0 for PASS, 1 for FAIL

Make the script executable with `chmod +x eval/e2e-assert.sh`.
  </action>
  <verify>
    <automated>which agent-browser && agent-browser --version && test -x eval/e2e-assert.sh && head -5 eval/e2e-assert.sh | grep -q 'agent-browser'</automated>
  </verify>
  <done>agent-browser is installed globally, `eval/e2e-assert.sh` exists and is executable with all 5 assertion types implemented</done>
</task>
