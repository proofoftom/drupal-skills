## Root Causes (4 bugs found)

1. **DashboardController: non-existent route** — `Url::fromRoute('group_ai_pm.project.add')` referenced a route that was never defined. Correct route is `entity.project.add_form` (auto-generated by AdminHtmlRouteProvider).

2. **DashboardController: wrong field name** — Entity query used `->condition('project_id', ...)` but the Task entity field is named `project`.

3. **KanbanController: wrong method target** — Called `$storage->getAccessControlHandler()` but storage doesn't have this method. Correct: `$this->entityTypeManager()->getAccessControlHandler('task')`.

4. **Libraries: Vue defer breaks IIFE** — `{ attributes: { defer: true } }` on Vue library caused kanban.js (IIFE consuming `Vue` global) to execute before Vue was defined.

## Fixes Applied

- `modules/group_ai_pm/src/Controller/DashboardController.php` — bugs 1 & 2
- `modules/group_ai_pm/src/Controller/KanbanController.php` — bug 3
- `modules/group_ai_pm/group_ai_pm.libraries.yml` — bug 4
- All synced to `drupal-10-group-ai-pm/web/modules/custom/group_ai_pm/`
