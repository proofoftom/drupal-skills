<context>
The user wants to push the limits of what Drupal + Claude skills can produce. v4.0 is about transforming a functional-but-bare module into something that could compete with Linear's UX quality. The eval-driven approach continues -- each phase measures skill delta with the addition of browser-based UX assertions (eval-browser) for frontend testing.

Key technical insight from research: the Vue app is an "island" embedded in Drupal's admin chrome, not a decoupled SPA. This means Drupal handles routing, auth, and the admin theme. Vue only owns the interactive Kanban board pages. Simpler interactions (status toggles, inline edits on list views) use Drupal's core AJAX framework.

The research identified CSRF token handling as the #1 integration gotcha (two different mechanisms in Drupal) and Haiku's declaration-usage gap as the likely dominant eval failure mode. Assertions must test WIRING between components, not just file existence.

Research is at .planning/research/ (STACK.md, FEATURES.md, ARCHITECTURE.md, PITFALLS.md, SUMMARY.md).
