**drupal-routing-controllers** (`skills/drupal-routing-controllers/evals/evals.json`):
- Module name: `api_status_endpoint`
- Prompt: Create a JSON API endpoint module for a Drupal 10 Open Social site (os-knowledge-garden). The module `api_status_endpoint` should provide a route at `/api/system/status` that returns a JSON response with site name and module count. The controller should use dependency injection to inject the module_handler and config.factory services. Use proper DI patterns with a create() factory method. We want D11 compatibility.
- Expected output: Controller extending ControllerBase with create() factory, DI services, JsonResponse, route with _permission
- Expectations (mix of tiers):
  1. [Static] Controller class extends ControllerBase (not bare ContainerInjectionInterface)
  2. [Static] Controller has static create() method accepting ContainerInterface
  3. [Static] Constructor uses typed service parameters with DI (not \Drupal::service() calls in methods)
  4. [Static] Route YAML uses _controller key with fully qualified class name
  5. [Static] Route has _permission or _access requirement (not unprotected)
  6. [Static] .info.yml has core_version_requirement including ^10 and ^11
  7. [Runtime] The module enables successfully: ddev drush en api_status_endpoint returns exit code 0
  8. [E2E] The API endpoint returns JSON: navigate to /api/system/status (as admin via drush uli) and verify page contains site name or module count data (not a 404 page)

**drupal-forms-api** (`skills/drupal-forms-api/evals/evals.json`):

Save all settings to config. Include a config schema file. We want D11 compatibility.
- Expected output: ConfigFormBase with getEditableConfigName, config schema, route with _form key
- Expectations (eval 1):
  1. [Static] Form class extends ConfigFormBase (not FormBase)
  2. [Static] Form class implements getEditableConfigNames() returning array with config name
  3. [Static] submitForm() uses $this->config() to save settings (not \Drupal::configFactory())
  4. [Static] Route YAML uses _form key (not _controller) for the settings route
  5. [Static] config/schema/{module}.schema.yml file exists with typed entries
  6. [Static] Schema types include integer or float types for numeric fields
  7. [Runtime] The module enables successfully: ddev drush en search_settings returns exit code 0
  8. [E2E] Settings form renders: navigate to /admin/config/search/search-settings (as admin) and verify form fields are present (page contains "Similarity" or "similarity_threshold")
