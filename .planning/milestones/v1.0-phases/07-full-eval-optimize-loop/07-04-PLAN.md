Run 3 skills through the full eval pipeline. For each skill, execute this sequence:

**Per-skill flow (do all 3 skills, one at a time to manage RAM):**

For SKILL in routing-controllers, forms-api, plugins-blocks:
1. Read the skill's evals.json to get prompt and eval name
2. Create workspace: `mkdir -p drupal-{SKILL}-workspace/iteration-1/eval-{name}/{with,without}_skill/run-1/outputs`
3. Setup 2 ddev envs: `bash eval/setup-drupal-env.sh {SKILL}-with` and `bash eval/setup-drupal-env.sh {SKILL}-without`
4. Run WITH skill (headless Sonnet):
   ```
   env -u CLAUDECODE claude -p --model sonnet --permission-mode bypassPermissions \
     "You are working in a Drupal project at /tmp/os-kg-{SKILL}-with. Read the skill file at /home/proofoftom/Code/drupal-skills/skills/drupal-{SKILL}/SKILL.md first, then: [EVAL PROMPT]. Create all files under /tmp/os-kg-{SKILL}-with/html/modules/custom/{module_name}/. After creating files, run: ddev drush en {module_name} -y" \
     > drupal-{SKILL}-workspace/iteration-1/eval-{name}/with_skill/run-1/transcript.md 2>&1
   ```
5. Copy outputs: `cp -r /tmp/os-kg-{SKILL}-with/html/modules/custom/{module_name}/ drupal-{SKILL}-workspace/iteration-1/eval-{name}/with_skill/run-1/outputs/`
6. Run WITHOUT skill (headless Sonnet) -- same prompt but WITHOUT the "Read the skill file" instruction
7. Copy outputs for without_skill
8. **Grade both runs:** For each run (with/without), evaluate generated code against each expectation in evals.json:
   - Static assertions: grep/check the output files
   - Runtime assertions: run drush commands against the ddev env
   - E2E assertions: use `eval/e2e-assert.sh` or agent-browser commands against the ddev site
   - Write grading.json with format: `{"expectations": [{"id": N, "description": "...", "passed": bool, "evidence": "..."}], "pass_rate": N/M}`
9. Teardown: `bash eval/teardown-drupal-env.sh {SKILL}-with && bash eval/teardown-drupal-env.sh {SKILL}-without`
10. Aggregate: `python3 aggregate_benchmark.py drupal-{SKILL}-workspace/iteration-1 --skill-name drupal-{SKILL}`

**Important:** Teardown each skill's environments before starting the next skill to stay within RAM limits. Do NOT run all 3 skills in parallel.

Module names per skill (from evals.json):
- routing-controllers: api_status_endpoint
- forms-api: search_settings
- plugins-blocks: content_recommendations
