**SKILL.md body (under 500 lines):**

1. **Decision tree: "What kind of route do you need?"**
   - Simple page (controller returns render array) -> basic route + controller
   - Form page -> route with `_form` default (defer to drupal-forms-api skill)
   - Entity page -> entity route provider handles it (defer to drupal-entities-fields skill)
   - Access-controlled page -> add `_permission` or `_custom_access` requirement
   - Admin page -> use admin theme via `_admin_route: TRUE` option
   - Route with parameters -> define parameter in path, optionally add `ParamConverter`

2. **.routing.yml complete reference**
   - Route structure: path, defaults (_controller or _form or _entity_form), requirements (_permission, _role, _access, _custom_access), options
   - Route parameters: `{parameter}` syntax, entity upcasting, type hints
   - WRONG-WAY CALLOUT: Using `hook_menu()` -- this is Drupal 7. Use .routing.yml since Drupal 8.
   - WRONG-WAY CALLOUT: Hardcoding access with `_access: 'TRUE'` for pages that should have permission checks

3. **Controller patterns**
   - Extending ControllerBase (provides helper methods: t(), redirect(), entityTypeManager(), etc.)
   - Return types: render arrays, Response objects, JsonResponse, RedirectResponse
   - WRONG-WAY CALLOUT: Returning plain strings -- always return render arrays for proper theming/caching
   - Title callbacks: static `_title` or dynamic `_title_callback`

4. **Services and dependency injection**
   - .services.yml structure: service name, class, arguments (@ prefix for service injection)
   - ContainerInjectionInterface: `create()` static method + constructor
   - ControllerBase already implements ContainerInjectionInterface
   - Injecting services into controllers: override `create()`, add constructor params
   - WRONG-WAY CALLOUT: `\Drupal::service()` inside controller/form/entity classes -- inject via create() + constructor. Static calls are ONLY acceptable in .module procedural code.
   - WRONG-WAY CALLOUT: Injecting the entire container -- inject specific services, not the container

5. **Complete route + controller + service example**
   - Show a realistic example with: .routing.yml route, Controller class with DI, custom service with its own DI, .services.yml binding them together
   - Show the full directory tree and every file

6. **Route access patterns** (brief)
   - `_permission`: check a single permission
   - `_role`: check a role
   - `_custom_access`: point to a method that returns AccessResult
   - Note: detailed access control in drupal-access-security skill

7. **Cross-references**
   - "See also: drupal-module-scaffold (if installed) for module creation and .info.yml setup."
   - "See also: drupal-entities-fields (if installed) for entity route providers that auto-generate CRUD routes."
   - "See also: drupal-forms-api (if installed) for form routes using _form default."
   - "For menu links, local tasks, and local actions, see references/menus.md in this skill directory."
