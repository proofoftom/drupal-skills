**Frontmatter (YAML):**
```yaml
---
name: drupal-module-scaffold
description: |
  Scaffold Drupal modules with correct .info.yml, PSR-4 namespace structure,
  and .module files. Use when asked to create a new Drupal module, start a
  custom module, or set up module boilerplate.
---
```

**Body structure (must be under 500 lines total including frontmatter):**

1. **Decision tree: "What files do you need?"**
   - ALWAYS create: module_name.info.yml, src/ directory
   - CREATE WHEN NEEDED: .module (only for hooks), .routing.yml (only for routes), .services.yml (only for services), .permissions.yml, .libraries.yml
   - Decision flow: "Creating a new module?" -> "Does it need routes?" -> "Does it need services?" etc.

2. **.info.yml complete reference**
   - Required keys: name, description, type, core_version_requirement, package
   - Optional keys: dependencies (project:module format), configure, lifecycle, hidden
   - WRONG-WAY CALLOUT: `core: 8.x` is Drupal 8 -- use `core_version_requirement: ^10 || ^11`
   - WRONG-WAY CALLOUT: dependencies as `module_name` alone -- use `project:module_name` format (e.g., `drupal:node`)

3. **PSR-4 namespace structure**
   - src/ maps to `Drupal\module_name\`
   - Standard subdirectories: Controller/, Form/, Entity/, Plugin/, EventSubscriber/, Service/
   - WRONG-WAY CALLOUT: Putting PHP classes outside src/ -- they won't autoload

4. **.module file patterns**
   - When to use: implementing hooks (hook_theme, hook_form_alter, hook_entity_type_build, etc.)
   - File structure: opening <?php, namespace documentation, use statements, hook implementations
   - WRONG-WAY CALLOUT: Using .module for class definitions -- classes go in src/ under PSR-4
   - WRONG-WAY CALLOUT: Using `\Drupal::service()` inside class methods -- use DI via create() + constructor (procedural .module code is the ONE place where `\Drupal::` calls are acceptable)

5. **Complete module scaffold example**
   - Show a realistic module (e.g., `hello_world`) with all key files
   - Include .info.yml, .module (with a hook_theme example), .routing.yml stub, .services.yml stub
   - Show the directory tree

6. **Cross-references** (at the end)
   - "See also: drupal-routing-controllers (if installed) for route definitions, controllers, and service injection patterns."
   - "See also: drupal-entities-fields (if installed) for content and config entity type creation."
   - "See also: drupal-forms-api (if installed) for Form API lifecycle and form altering."

Extract all technical content from the book (Ch 1-2). Key topics from the book:
- Module folder goes in `modules/custom/` or `modules/contrib/`
- .info.yml keys and their meaning
- The hook system and .module file
- PSR-4 autoloading in Drupal
- Services and the service container overview (detail deferred to routing skill)

D10/D11 note: Module scaffolding has minimal D10/D11 differences (no annotations/attributes involved). Note that `core_version_requirement: ^10 || ^11` supports both versions. Mention that D11 is moving toward PHP attributes for plugins (relevant when the module will define plugins, covered by other skills).
