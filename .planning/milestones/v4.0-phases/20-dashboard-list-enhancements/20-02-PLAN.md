Create `src/Form/TaskStatusForm.php` extending `FormBase` with dependency injection:

**Namespace:** `Drupal\group_ai_pm\Form`

**Use statements:**
- `Drupal\Core\Entity\EntityTypeManagerInterface`
- `Drupal\Core\Datetime\DateFormatterInterface`
- `Drupal\Core\Form\FormBase`
- `Drupal\Core\Form\FormStateInterface`
- `Symfony\Component\DependencyInjection\ContainerInterface`

**DI pattern:** Inject `EntityTypeManagerInterface` and `DateFormatterInterface` via `create()` factory and constructor (NOT `$this->entityTypeManager()` lazy method).

**getFormId():** Return `'group_ai_pm_task_status_form'`.

**buildForm():**
1. Query tasks: `$this->entityTypeManager->getStorage('task')->getQuery()->accessCheck(TRUE)->sort('created', 'DESC')->pager(50)->execute()`.
2. Load tasks: `$this->entityTypeManager->getStorage('task')->loadMultiple($task_ids)`.
3. Build table header: Title, Project, Status, Priority, Due Date, Assignee.
4. Create `$form['tasks']` as `#type => 'table'` with `#header` and `#empty => $this->t('No tasks found.')`.
5. For each task (keyed by entity ID):
   - `$form['tasks'][$task_id]['title']`: `#markup` with `$task->getTitle()`.
   - `$form['tasks'][$task_id]['project']`: `#markup` with project title (check `$task->get('project')->entity` exists).
   - `$form['tasks'][$task_id]['status']`: `#type => 'select'` with:
     - `#options => ['todo' => $this->t('To Do'), 'in_progress' => $this->t('In Progress'), 'review' => $this->t('Review'), 'done' => $this->t('Done')]`
     - `#default_value => $task->getStatus()`
     - `#ajax => ['callback' => '::statusUpdateCallback', 'event' => 'change', 'wrapper' => 'task-row-' . $task_id]`
   - `$form['tasks'][$task_id]['priority']`: `#markup` with `$task->getPriority()`.
   - `$form['tasks'][$task_id]['due_date']`: `#markup` with formatted due date (use `$this->dateFormatter->format(strtotime($due_date), 'short')` if due date exists, empty string otherwise).
   - `$form['tasks'][$task_id]['assignee']`: `#markup` with assignee display name (check `$task->get('assignee')->entity` exists).
   - Set row wrapper ID: `$form['tasks'][$task_id]['#attributes']['id'] = 'task-row-' . $task_id`.
6. Add pager: `$form['pager'] = ['#type' => 'pager']`.
7. Return `$form`.

**statusUpdateCallback():**
1. Get triggering element: `$form_state->getTriggeringElement()`.
2. Extract task ID from `$trigger['#array_parents'][1]`.
3. Get new status: `$form_state->getValue(['tasks', $task_id, 'status'])`.
4. Load task: `$this->entityTypeManager->getStorage('task')->load($task_id)`.
5. If task exists: `$task->setStatus($new_status)` then `$task->save()`.
6. Return `$form['tasks'][$task_id]` (the rebuilt row).

**submitForm():** No-op -- status changes happen via AJAX callbacks. Add a docblock comment explaining this: "No-op: status changes are handled by AJAX callbacks."

CRITICAL: The `#ajax['wrapper']` value MUST match the row's `#attributes['id']` exactly. Both use `'task-row-' . $task_id`. If these don't match, AJAX will silently fail.

```yaml
  path: '/admin/content/task/overview'
  defaults:
    _form: '\Drupal\group_ai_pm\Form\TaskStatusForm'
    _title: 'Task Overview'
  requirements:
    _permission: 'access group_ai_pm dashboard'
  options:
    _admin_route: TRUE
```

Key points:
- Uses `_form` key (NOT `_controller`) because TaskStatusForm extends FormBase. The `_form` route key ensures Drupal's form processing pipeline handles the form, including AJAX processing, form state management, and CSRF token generation.
- Uses `_admin_route: TRUE` so it renders within the admin theme (Claro).
- Uses the same permission as the dashboard and kanban board routes for consistency.
- Path `/admin/content/task/overview` is separate from the entity collection route (`entity.task.collection`) to avoid breaking existing entity CRUD routes.

Do NOT modify any existing routes. Only append this new route.
