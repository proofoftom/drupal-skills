Create `TaskApiController.php` extending ControllerBase with DI for entity_type.manager and current_user. Include these methods:

1. `kanban(ProjectInterface $project)` (API-01, API-06, API-08):
   - Check `$project->access('view')`, throw AccessDeniedHttpException if denied
   - Query task storage: `condition('project', $project->id())`, `accessCheck(TRUE)`, sort by priority DESC, due_date ASC, created DESC
   - Load tasks, filter each through `$task->access('view')`, group into `$columns = ['todo' => [], 'in_progress' => [], 'review' => [], 'done' => []]`
   - Serialize each task via private `serializeTask()` method returning: id (int), title, status, priority, assignee ({id, name} or null), dueDate, created (int), changed (int), editUrl
   - Return `CacheableJsonResponse` with `['project' => ['id' => ..., 'title' => ...], 'columns' => $columns]`
   - Add CacheableMetadata with tags `['task_list', 'project:' . $project->id()]`, context `['user.permissions']`
   - Add each loaded entity as cacheable dependency

2. `updateStatus(TaskInterface $task, Request $request)` (API-02, API-06):
   - Check `$task->access('update')`, throw AccessDeniedHttpException if denied
   - Decode JSON body, validate `status` field exists and is in allowed values (todo, in_progress, review, done)
   - Return 422 JsonResponse if invalid
   - Set status, save entity
   - Return `CacheableJsonResponse` with serialized task, add entity as cacheable dependency

3. `updateTask(TaskInterface $task, Request $request)` (API-03, API-06):
   - Check `$task->access('update')`, throw AccessDeniedHttpException if denied
   - Decode JSON body, accept optional fields: title (string 1-255), priority (allowed values), assignee (user ID or null)
   - Validate each provided field, return 422 if invalid
   - If assignee provided, load user entity and verify it exists
   - Update fields, save entity
   - Return `CacheableJsonResponse` with serialized task

4. `quickCreate(ProjectInterface $project, Request $request)` (API-04, API-06):
   - Check `$project->access('view')` AND check createAccess on task storage
   - Decode JSON body, require `title` (string 1-255), optional `status` (defaults to 'todo')
   - Validate status in allowed values
   - Create task entity with: title, status, project reference, uid = current user, priority = 'medium' default
   - Save and return 201 `JsonResponse` (not CacheableJsonResponse -- new entities should not be cached) with serialized task

Private helper `serializeTask(TaskInterface $task)`:
   - Return array: `id` => (int), `title`, `status`, `priority`, `assignee` => (entity loaded from field, {id, name} or null), `dueDate`, `created` => (int), `changed` => (int), `editUrl` => toUrl('edit-form')->toString()

Create `ProjectApiController.php` extending ControllerBase with these methods:

5. `summary(ProjectInterface $project)` (API-05, API-06, API-08):
   - Check `$project->access('view')`, throw AccessDeniedHttpException if denied
   - Query task count per status using 4 separate count queries (or single query grouped)
   - Return `CacheableJsonResponse` with `['project' => ['id', 'title', 'status'], 'taskCounts' => ['todo' => N, 'in_progress' => N, 'review' => N, 'done' => N], 'totalTasks' => N]`
   - Add CacheableMetadata with tags `['task_list', 'project:' . $project->id()]`

CRITICAL: Use `CacheableJsonResponse` from `Drupal\Core\Cache\CacheableJsonResponse` and `CacheableMetadata` from `Drupal\Core\Cache\CacheableMetadata`. These are core classes, no extra modules needed.

CRITICAL: Do NOT use `\Drupal::` static calls. Use dependency injection via `create()` + `__construct()` pattern (ContainerInterface). Inject `entity_type.manager` and `current_user`.
