## Bugs Fixed This Session
1. **Drag-and-drop not updating status** — `@change`/`e.moved` replaced with `@add` (SortableJS event). vue-draggable-plus v0.1.6 fires `@add` on destination column for cross-list drops, NOT `@change` with `{added, moved, removed}` (that's the old vuedraggable library).
   - Files: `KanbanColumn.vue` (handleDragAdd)
2. **Panel save not persisting status** — Backend `updateTask()` only handles title/priority/assignee. Status has a dedicated endpoint (`/api/kanban/task/{id}/status`). `handlePanelUpdate` in KanbanBoard.vue now splits status to `updateTaskStatus()` and remaining fields to `updateTask()`.
   - Files: `KanbanBoard.vue` (handlePanelUpdate)
3. **Panel showing stale data** — Watcher on `props.task` didn't fire when same object reference was passed with mutated properties. Added `deep: true`.
   - Files: `TaskDetailPanel.vue`
4. **Panel not closing on save** — Added `emit('close')` after emitting update-task.
   - Files: `TaskDetailPanel.vue`
5. **Quick create ignoring column status** — Status threaded through entire chain: QuickCreateForm → KanbanColumn (already passed) → KanbanBoard.handleTaskCreate → useKanban.createTask → backend quickCreate. Backend now accepts `status` from request body, validates against allowed list, defaults to 'todo'.
   - Files: `KanbanBoard.vue`, `useKanban.js`, `TaskApiController.php`

<decisions_made>

- Used `@add` SortableJS event instead of `@change` for cross-column drag detection — `@change` in vue-draggable-plus v0.1.6 is a raw SortableJS event with `{newIndex}`, NOT the vuedraggable-style `{added, moved, removed}` object
- Split status updates to dedicated endpoint rather than adding status handling to the general updateTask endpoint — keeps concerns separated and avoids touching the established backend pattern
- Deep watcher on task prop rather than cloning the task object — simpler, though slightly less efficient
- Reverted symlink to copy approach — ddev container mount boundary makes symlinks impractical without docker-compose overrides
- Created Vue reference as separate file under drupal-theming skill (not a new skill) — Vue patterns are theming-adjacent and don't warrant a standalone skill
